# 거지AI — "무료 AI 서비스, 한 곳에서 찾고 비교하자"

<p style="font-size: 52px; line-height: 1; margin: 0 0 12px;">🤖</p>

**만든 이: 종인**

## 01 시작은 "왜 한국어로 된 AI 도구 모음 사이트가 없지?"였다

2026년 초, AI 도구가 매일같이 쏟아져 나오고 있었다. ChatGPT, Claude, Gemini, Copilot... 해외에는 TAAFT(There's An AI For That) 같은 80만 사용자, 49,000개 툴을 모아놓은 거대 디렉토리가 있었다. 하지만 한국어로 된, 한국 사용자를 위한 AI 도구 모음 사이트는 변변한 게 없었다.

"내가 직접 만들면 되지 않을까?"

이 생각이 들었다. 문제는, 필자는 웹 개발 전문 개발자가 아니었다는 것. 하지만 Vibe Coding의 시대. 말로 설명할 수 있다면 만들 수 있다. 그래서 시작했다.

## 02 "말"로 만든 거지AI

거지AI( BrokeAI )의 컨셉은 심플했다:

> **한국 사용자가 무료 AI 도구를 가장 쉽게 발견하고 비교할 수 있는 곳**

기획은 이랬다:
- 🔍 **검색 + 카테고리**: 원하는 AI 툴을 빠르게 찾기
- 💰 **무료 티어 정보**: "이거 진짜 무료야?"를 한눈에
- ⭐ **별점 + 리뷰**: 실제 사용자들의 평가
- ⚖️ **비교 도구**: 2-3개 툴을 나란히 비교
- 🧠 **AI 추천 퀴즈**: 5개 질문으로 맞춤 툴 추천
- 💼 **직업별 추천**: 개발자, 디자이너, 마케터... 직업에 맞는 AI 툴

이 모든 걸 AI 코딩 에이전트와 "대화"하면서 만들었다. "무료 필터를 3단계로 나눠줘 — 완전무료, 부분무료, 체험만", "태그를 클릭하면 필터링되게 해줘", "비교 트레이를 하단에 띄워줘" — 이렇게 말하면 코드가 만들어졌다.

## 03 기술 스택: 심플하게, 그러나 강력하게

| 구성 요소 | 기술 |
|-----------|------|
| 프레임워크 | Next.js App Router |
| 데이터베이스 | SQLite (better-sqlite3) |
| 스타일 | Tailwind CSS |
| 인증 | NextAuth v5 (Google OAuth) |
| 검색 | FTS5 전문검색 |
| 호스팅 | 자체 서버 (pm2 + nginx) |

Prisma 같은 ORM도 안 쓰고, SQLite 하나로 55개 서비스, 9개 카테고리, 전문검색, 트렌딩 로직, 수정 이력까지 다 처리했다. "복잡하게 갈 필요 없다"는 철학이었다.

## 04 경쟁사 분석으로 차별화 포인트 찾기

글로벌과 한국 경쟁사를 분석했다:

- **TAAFT**: 49K 툴, 무료필터 3단계, ⌘+K 검색
- **AI모아 (한국)**: 직업별 분류, 모아스코어 5항목×5점, 한국어 지원 평가
- **AI Tools Scoop**: 월간/연간 가격 비교, side-by-side 비교
- **AIpedia 한국**: 별점 랭킹 TOP 20

분석 결과, 한국 시장에서 **아무도 가격 추적, 비교 도구, 한국어 지원 평가**를 하고 있지 않다는 걸 발견했다. 이게 거지AI의 차별점이 되었다.

## 05 GSD + Superpowers 워크플로우

18개 기능을 3 Phase로 나눠서 진행했다:

**Phase 1 — Quick Wins (3-5일)**
- 무료 필터 세분화 (완전무료/부분무료/체험만)
- 검증일 뱃지
- 태그 클릭 필터링
- AdSense 최적 배치
- sitemap.xml + robots.txt
- OG 메타데이터
- 404 + loading 스켈레톤

**Phase 2 — Core Differentiation (1-2주)**
- Side-by-side 비교 도구 (비교 트레이 + 매트릭스)
- "X 대안" 자동 생성 페이지 (프로그래매틱 SEO)
- 직업별 추천 (8개 직업)
- 별점 시스템 + TOP 20 랭킹
- 북마크 "내 AI 도구함"
- 한국어 지원 레벨 (1-5점)

**Phase 3 — Growth Engine (2-4주)**
- 무료티어 변경 이력 타임라인
- 유저 리뷰 + 인증 뱃지
- AI 매칭 퀴즈 (5질문 → 맞춤 추천)
- 네이버 SEO (JSON-LD + 인증 메타)

## 06 결과물: 40개 라우트, 18개 기능, 0개 TypeScript 에러

모든 걸 만들고 나니, Next.js 빌드가 40개 라우트를 생성했다:

```
/                    홈 (검색 + 필터 + 카드 그리드)
/compare             코딩 에이전트 비교 + 비교 매트릭스
/for/[role]          직업별 추천 (8개 직업)
/alternatives/[id]   "X 대안" 페이지
/quiz                AI 매칭 퀴즈
/ranking             별점 랭킹
/my-tools            북마크 "내 AI 도구함"
/services/[id]       서비스 상세 + 리뷰 + 변경이력
/sitemap.xml         자동 생성
/robots.txt          자동 생성
```

배치 작업도 자동화했다:
- 매시간: 트렌딩 스코어 갱신
- 매일 02:30: 서비스 URL 헬스체크 + 검증일 갱신
- 매일 04:00: SQLite DB 백업 (30일 보관)

## 07 "Vibe Coding, 진짜 가능하더라"

솔직히 말하면, 처음엔 반신반의했다. "말로 만든다?" 그게 될까? 하지만 실제로 해보니, AI 코딩 에이전트는 정말로 내 말을 이해하고 코드를 만들어냈다.

물론 문제도 있었다:
- 빌드는 성공하는데 실제 동작이 깨진 적도 있었다 (`.next` 캐시 꼬임)
- `SessionProvider` 누락으로 SSR 에러가 난 적도 있었다
- 여러 subagent가 동시에 DB 스키마를 변경하려다 충돌할 뻔한 적도 있었다

하지만 핵심은 이거다: **기획만 확실히 하면, 코딩은 AI가 해준다.** 필자가 한 건 "무엇을 만들지"를 정하는 것이었고, AI가 "어떻게 만들지"를 해결했다.

## 08 거지AI를 만들며 배운 것

1. **기획이 80%, 코딩이 20%**: 무엇을 만들지 명확히 아는 게 코드 작성보다 중요하다
2. **경쟁사 분석은 필수**: 남들이 안 하는 걸 찾는 게 차별화의 핵심이다
3. **빌드 성공 ≠ 동작 성공**: 실제 렌더링, 에러 로그, 크롤링까지 확인해야 한다
4. **단순함이 힘이다**: SQLite 하나로 충분하다. 복잡한 인프라는 필요 없다
5. **Vibe Coding은 진짜다**: 개발자가 아니어도, 아이디어가 있다면 만들 수 있다
